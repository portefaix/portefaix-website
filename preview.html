06 // Guardrails

Do's and Don'ts

✓ DO
Use #0ea5e9 only for interactive / K8s elements
Use monospace for system-level text: repo names, section IDs, status labels
Keep section IDs in format 01 // Name
Use shadows on light mode; use bg color difference on dark mode for elevation
Keep per-project colors isolated — violet=TF, rose=Hub, amber=Policies
Test both dark and light themes for every new component
✗ DON'T
Use font-weight 800 below h1 size — illegible at small sizes
Introduce a third typeface — Inter + JetBrains Mono is the full set
Use opacity on colored text for hierarchy — use text.secondary/muted tokens
Duplicate the scan-line gradient outside the hero section
Stack multiple glow shadows — one glow per element maximum
Use status.error red for non-error emphasis or branding
Animate layout properties (width/height/padding) — only transform + opacity
Use hardcoded spacing — always use 4px-base scale multiples
